# 异常检测算法 — 实验室版 vs 生产环境

> 2026-06-26 | 当前代码为实验室水池调试版

---

## 一、SC 船-船搭靠

### 实验室版（当前代码）

位置：`anomaly_detector.py` 第 272-374 行

```
两两配对，每帧检查：
  ① 两船距离进入 1m 范围，开始追踪"接近过程"
  ② 距离下降 ≥0.3m（证明主动靠近）
  ③ 距离 <0.5m + 相对速度 <0.3m/s（证明并靠，非擦肩而过）
  ④ 以上状态持续 ≥5 秒 → 报警
```

配置：`live_config.json` → `ship_rendezvous`

| 参数 | 实验室值 | 含义 |
|------|---------|------|
| `distance_threshold_m` | 0.5 | 两船距离多近算"贴在一起" |
| `duration_threshold_s` | 5.0 | 持续多久才报警（测试用，正式实验 15 秒） |
| `relative_speed_threshold_m_s` | 0.3 | 两船速度差上限。**核心防误报**：墙壁静止、船移动 → 相对速度 0.7>0.3 → 不报 |
| `approach_distance_threshold_m` | 1.0 | 多远开始追踪接近过程。水池只有 4m 长 |
| `min_distance_drop_m` | 0.3 | 接近过程距离至少下降多少。排除恰好路过的两个噪声 |

**实验室改动点：** `duration` 从 15s 缩短到 5s（加速测试），`approach_distance` 从 2m 缩小到 1m（水池空间限制）。

### 生产环境版

所有逻辑不变。参数调整：

| 参数 | 生产值 | 原因 |
|------|--------|------|
| `duration_threshold_s` | **15.0** | 实验方案文档要求 |
| `approach_distance_threshold_m` | **10~50m** | 港口尺度远大于水池 |
| `distance_threshold_m` | **2~5m** | 真实船舶尺寸大，靠泊距离也大 |
| `min_distance_drop_m` | **5~20m** | 港口尺度下的合理接近距离下降 |

## 二、KS 快速通过

### 实验室版（当前代码）

位置：`anomaly_detector.py` 第 380-437 行

```
每个目标独立判断：
  ① 速度 >0.5m/s
  ② 持续 ≥0.3 秒
  ③ 报警
  （surge/accel 条件已关闭）
```

配置：`live_config.json` → `fast_passage`

| 参数 | 实验室值 | 含义 |
|------|---------|------|
| `speed_threshold_m_s` | 0.5 | 遥控船极速约 1.0m/s。设 0.5 让船加速到一半就能触发达 |
| `min_duration_s` | 0.3 | 至少持续多久。雷达约 10~20Hz，0.3 秒 ≥3 帧，排除瞬时毛刺 |
| `speed_ratio_threshold` | 999 | **已禁用**（实验室调试阶段只靠绝对速度） |
| `acceleration_threshold_m_s2` | 999 | **已禁用** |

**实验室改动点：** `speed_threshold` 降到 0.5（小船极速才 1m/s），`surge/accel` 关掉（简化触发条件，方便验证基础逻辑）。

### 生产环境版

恢复 surge/accel 判断，参数调整为港口尺度：

| 参数 | 生产值 | 原因 |
|------|--------|------|
| `speed_threshold_m_s` | **10~15** | 港口快艇速度。真实阈值取决于场景（航道限速等） |
| `speed_ratio_threshold` | **2.0~3.0** | 恢复启用。区分"本来就快的船"和"突然变快的船" |
| `acceleration_threshold_m_s2` | **1.0~2.0** | 恢复启用。快艇急加速远比遥控船大 |
| `min_duration_s` | **2.0** | 恢复原值，过滤瞬时跳变 |

## 三、KA 船靠岸送人上岸

### 实验室版（当前代码）

位置：`anomaly_detector.py` 第 440-600 行

双路径，任一触发即报警：

**路径A（偏航+减速停留）：**

```
① 目标必须曾经移动过（was_moving=True）——排除墙壁固定反射
② X 偏离航道中心线 >0.3m（偏航）
③ 速度 <0.2m/s（减速停止）
④ 持续 ≥10 秒 → 报警
```

**路径B（运动特征，不依赖区域多边形）：**

```
① 距 unauthorized_docking_zone 边界线 <0.5m 算"到达岸边"
② 回溯 20 秒内的运动：接近速度 >0.3m/s + 距离下降 >0.3m → 确认快速接近
③ 到达后速度 <0.2m/s + 持续 ≥10 秒 → 报警
```

配置：`live_config.json` → `shore_docking`

| 参数 | 实验室值 | 含义 |
|------|---------|------|
| `deviation_threshold_m` | 0.3 | 航道宽 0.6m，中心 ±0.3m 合理 |
| `speed_threshold_m_s` | 0.2 | 多慢算"停止" |
| `duration_threshold_s` | 10.0 | 停了多久算"靠岸" |
| `shore_arrival_distance_m` | 0.5 | 距岸边多远算"到了" |
| `shore_fast_approach_mps` | 0.3 | 接近阶段的平均速度下限 |
| `shore_min_distance_drop_m` | 0.3 | 接近阶段距离下降下限 |

**实验室改动点：** 路径A 加了 `was_moving` 保护，防止墙壁反射误触。这是实验室特有的问题——密闭房间的固定反射在海面上不存在。

### 生产环境版

生产环境不依赖 `unauthorized_docking_zone` 多边形（路径B 用），因为海面没有固定岸线多边形。改为靠 AOI（电子围栏区域）+ 路径A 的逻辑。

能搬过去的：
- 路径A 的状态机逻辑（偏航→减速→停留）
- 路径B 的运动特征判断逻辑（接近速度 + 距离下降 + 到达后低速）

要改的：
- `normal_channel` 改为 GPS 经纬度标注的航道边界
- `unauthorized_docking_zone` 改为 AOI 敏感区域定义（如某个码头段）
- 速度/距离阈值放大到港口尺度

## 四、与生产环境的差异总表

| 维度 | 实验室版 | 生产环境版 |
|------|---------|-----------|
| **SC 接近距离** | 1m | 10~50m |
| **SC 贴靠距离** | 0.5m | 2~5m |
| **SC 持续时间** | 5s（测试） | 15s（规范） |
| **KS 速度阈值** | 0.5m/s | 10~15m/s |
| **KS surge/accel** | 已关闭 | 启用 |
| **KA 防静止误报** | `was_moving` 保护 | 不需要（海面无固定墙壁反射） |
| **坐标系统** | 雷达极坐标→米 | GPS 经纬度 |
| **区域定义** | JSON 多边形顶点 | GIS 电子围栏 |
| **滤波重点** | 墙壁/水面/天花板反射 | 海杂波 |
| **雷达参数** | 室内短距（0.3~4m） | 海面远距（50m~5km） |
