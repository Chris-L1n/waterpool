## 二、会产生哪些 CSV 文件

程序运行后，在项目目录下自动创建 `csv文件/` 文件夹。每次运行生成一个以启动时间命名的子文件夹，里面放 4 个 CSV 文件：

```
csv文件/
└── “当前时间” 比如（20260626_143025）/          ← 启动时间（年月日_时分秒）
    ├── radar_targets.csv     ← 全部雷达目标
    ├── radar_nearest.csv     ← 最近目标
    ├── radar_tracks.csv      ← 带航迹的全部目标
    └── anomaly_events.csv    ← 异常报警记录
```

### radar_targets.csv — 全部雷达目标

每帧每个目标都有一行。由 `radar_track_parser_v4.py` 从雷达原始数据解析并写入。

| 列名 | 含义 | 示例 |
|------|------|------|
| `pc_time` | 本机记录时间 | `2026-06-26T14:30:25.123` |
| `packet_no` | 雷达包序号（第几个有效的目标输出包） | `142` |
| `target_index` | 当前帧内的目标临时编号（0, 1, 2...） | `0` |
| `track_id` | 跨帧稳定跟踪ID（SimpleTracker 最近邻匹配分配） | `3` |
| `match_distance_m` | 当前目标与匹配轨迹上一帧位置的距离，新目标为 0 | `0.012` |
| `x_cm` | 雷达原始 X 坐标，单位厘米，雷达右侧为正 | `-25` |
| `y_cm` | 雷达原始 Y 坐标，单位厘米，雷达前方为正 | `340` |
| `speed_cm_s` | 雷达原始速度，单位厘米/秒，远离雷达为正 | `30` |
| `pv` | 雷达信号质量 / 信噪比，数值越大信号越好 | `52` |
| `x_m` | X 坐标，单位米（x_cm / 100） | `-0.25` |
| `y_m` | Y 坐标，单位米（y_cm / 100） | `3.40` |
| `speed_m_s` | 速度，单位米/秒（speed_cm_s / 100） | `0.30` |
| `distance_m` | 目标距雷达的欧氏距离，单位米 | `3.41` |

### radar_nearest.csv — 最近目标

每帧只记录**一个**目标：通过全部过滤条件且距离雷达最近的那个。**13列，列名与 `radar_targets` 完全相同。** 用于回顾"摄像头本来应该跟踪哪个目标"。

### radar_tracks.csv — 带航迹的全部目标

**13列，列名与 `radar_targets` 完全相同，数据也相同。** 为后续航迹分析（按 `track_id` 分组查看完整轨迹）预留。

### anomaly_events.csv — 异常报警记录

异常检测算法触发报警时写入，每行一次报警事件。**只有检测到异常时才有内容。**

| 列名 | 含义 | 示例 |
|------|------|------|
| `event_time` | 报警触发时间 | `2026-06-26T14:31:02.456` |
| `event_type` | 异常类型代码（SC/KS/KA） | `SC` |
| `event_label` | 异常类型中文标签 | `船-船搭靠` |
| `track_ids` | 涉及的目标 track_id，多个用逗号分隔 | `1,2` |
| `duration_s` | 异常持续时长，单位秒 | `15.0` |
| `details` | 详细信息，JSON 格式（含距离、速度、位置、触发路径等） | `{"distance_m":0.112,...}` |

## 三、CSV文件

新csv文件不会覆盖上一次csv。 

每次启动自动创建新的 `csv文件/”当前时间“/` 文件夹，旧文件夹自动保留。 

用时间戳区分每次的记录

## 四、怎么看有没有效果

### 4.1 程序跑起来的时候

终端出现这样的输出就说明检测到了：

```
==================================================
[ALARM #1] [SC] 船-船搭靠
  track_ids: [1, 2]
  duration: 15.0s
  details: {"distance_m": 0.112, ...}
==================================================
```

没异常时终端不会打印任何报警信息。

### 4.2 程序结束后

打开对应时间文件夹下的 `anomaly_events.csv`。空文件 = 没检测到。有行 = 检测到了。
