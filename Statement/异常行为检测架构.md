# 异常行为检测 — 架构说明

> 2026-06-25 | 供小组全员阅读

---

## 一、系统总览

```
                      雷达
                       │
                  CAN 帧 (0x421)
                       │
                       ▼
              radar_track_parser_v4.py
              ├─ CAN组包（拼成完整雷达包）
              ├─ 协议解析（提取 X/Y/速度/PV）
              ├─ 目标跟踪（分配跨帧稳定的 track_id）
              └─ CSV 写入（radar_targets.csv, radar_tracks.csv）
                       │
                       ▼ targets = [{track_id, x_m, y_m, speed_m_s}, ...]
                       │
                       ▼
              anomaly_detector.py    ← 本文件
              │
              ├── 无异常 → 系统静默
              │
              └── 检测到异常 → 
                      ├─ 写 anomaly_events.csv
                      ├─ 终端打印报警
                      └─ 回调 on_anomaly(event)
                            │
                            ▼
                      live_tracking.py（异常响应）
                            │
                            摄像头转向对应预置位
                            截图取证 → 回全局观察位
```

**异常检测是系统的核心调度者。没有异常时系统静默，检测到异常时才触发报警和摄像头动作。**

## 二、三类异常行为

| 编号 | 名称 | 场景 | 关键判定条件 |
|------|------|------|-------------|
| SC | 船-船搭靠 | 两船并靠（模拟非法过驳） | 两船距离 < 0.5m 且持续 ≥ 15秒 |
| KS | 快速通过 | 船只突然加速冲关 | 速度 > 1.5m/s 且明显加速（突增2倍或加速度异常） |
| KA | 船靠岸送人上岸 | 偏航驶向岸边后停留 | 偏离航道 → 进入禁区 → 减速 → 岸边停留 ≥ 10秒 |

## 三、三个核心算法

### 算法1：CAN组包（RadarReassembler）
- 文件：`radar_track_parser_v4.py`
- 功能：把零散的CAN帧拼成完整雷达数据包
- 输入：CAN帧（每帧8字节）→ 输出：完整雷达包

### 算法2：目标跟踪（SimpleTracker）
- 文件：`radar_track_parser_v4.py`
- 功能：给每帧的目标分配稳定的 track_id（雷达本身不给ID）
- 方法：最近邻匹配（两帧之间距离最近的是同一个目标）
- 用途：**异常检测依赖这个ID**（区分不同目标是搭靠检测、靠岸状态机的基础）

### 算法3：异常检测（AnomalyDetector）
- 文件：`anomaly_detector.py`
- 功能：实时检测三类异常行为
- 三种检测独立运行，各自有冷却时间防重复报警
