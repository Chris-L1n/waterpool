## 二、三类异常行为

| 编号 | 名称 | 场景 | 关键判定条件 |
|------|------|------|-------------|
| SC | 船-船搭靠 | 两船并靠（模拟非法过驳） | 两船距离 < 0.5m 且持续 ≥ 15秒 |
| KS | 快速通过 | 船只突然加速冲关 | 速度 > 1.5m/s 且明显加速（突增2倍或加速度异常） |
| KA | 船靠岸送人上岸 | 偏航驶向岸边后停留 | 偏离航道 → 进入禁区 → 减速 → 岸边停留 ≥ 10秒 |

实验方案详见：`文档/水池条件下三类异常行为识别实验方案（剧本）.docx`

## 三、三个核心算法

### 算法1：CAN组包（RadarReassembler）
- 文件：`radar_track_parser_v4.py`
- 功能：把零散的CAN帧拼成完整雷达数据包
- 输入：CAN帧（每帧8字节）→ 输出：完整雷达包

### 算法2：目标跟踪（SimpleTracker）
- 文件：`radar_track_parser_v4.py`
- 功能：给每帧的目标分配稳定的 track_id（雷达本身不给ID）
- 方法：最近邻匹配（两帧之间距离最近的是同一个目标）
- 用途：**所有下游都依赖这个ID**（CSV、PTZ、异常检测）

### 算法3：异常检测（AnomalyDetector）
- 文件：`anomaly_detector.py`
- 功能：实时检测三类异常行为
- 三种检测独立运行，各自有冷却时间防重复报警

## 五、异常检测算法细节

### SC 船-船搭靠

```
当前帧所有目标 → 两两算距离
  ├─ 距离 < 0.5m + 两船均在搭靠区 → 开始计时
  ├─ 持续 ≥ 15s → 报警 "SC 船-船搭靠"
  └─ 任一条件不再满足 → 计时清零
```

### KS 快速通过

```
每个目标逐一检查：
  ├─ 不在快速通过区？→ 跳过
  ├─ 速度 < 1.5m/s？→ 跳过
  ├─ 速度 ≥ 历史平均速度的2倍？→ 报警 "KS 快速通过"
  └─ 或：加速度 > 0.3 m/s²？→ 报警 "KS 快速通过"
```

### KA 船靠岸送人上岸（状态机）

```
NORMAL ──(偏航)──→ DEVIATED ──(进禁区)──→ IN_ZONE ──(减速)──→ STOPPED
                                                                   │
                                                          停留 ≥ 10秒
                                                                   │
                                                                   ▼
                                                              报警 "KA"
任一阶段条件不满足 → 立即回到 NORMAL（状态机重置）
```

## 六、原型代码参考：detect_anomalies.py

在 `test_3 - 副本/recognize/` 下有一份**离线批处理版**的异常检测原型。它是为港口场景（经纬度坐标）写的，读完整CSV后一次性分析。三种检测逻辑如下：

### 原型 SC — 两船搭靠 (detect_berthing)

```
① 对所有船两两配对，按时间戳对齐（pd.merge_asof）
② 两船距离 < 阈值 + 相对速度 < 阈值 → 标记为"正在并靠"
③ 并靠片段持续 ≥ 阈值 → 满足基础条件
④ 回溯并靠前的"接近窗口"，计算距离下降量
   距离下降量 ≥ 阈值 → 确认"两船在靠近而非恰好经过" → 报警
```

**比当前版本多做了什么**：相对速度约束 + 接近窗口距离下降量，排除了"两船高速擦肩而过"的误报。

### 原型 KS — 快速通过 (detect_high_speed)

```
① 每个目标单独分析
② 速度 ≥ 阈值 + 在港区内 → 标记为超速
③ 连续超速片段持续 ≥ 最小时长 → 报警
```

**与当前版本的区别**：原型只判断"超速+持续"，更简单。当前版本多了加速度和速度突增倍率判断。

### 原型 KA — 船靠岸 (detect_shore)

```
① 完全不依赖区域多边形！
② 计算每个点到"岸边"的距离（shore_dist）
③ 找到目标首次进入岸边距离阈值的时刻（arrival）
④ 回溯到达前的"接近窗口"：
   接近速度 ≥ 阈值 + 距离下降量 ≥ 阈值 → 快速接近了岸边
⑤ 到达后速度 ≤ 阈值 → 到达后停在岸边 → 报警
```

**与当前版本的区别**：原型完全不依赖区域定义，纯靠运动特征（接近速度 + 距离下降 + 到达后低速）判断。这是当前版本最缺的能力。

### 原型检测优先级与去重

```
检测顺序：搭靠 → 靠岸 → 快速通过
去重规则：靠岸命中的目标，不再报"快速通过"（靠岸本身就包含快速接近岸边的运动）
```

---

### 为什么不能直接搬？

| 差异点 | 原型 | 生产环境 | 影响 |
|--------|------|----------|------|
| **运行模式** | 读完整CSV → 一次性分析 | 每帧 `feed()` 实时检测 | 架构不兼容 |
| **依赖** | `pandas` + `numpy`（额外安装，~50MB） | 纯 Python 标准库 | 增加部署复杂度 |
| **坐标系** | 经纬度 → 米转换（港口） | 雷达原始坐标系，就是米（水池） | 坐标转换函数冗余 |
| **输入格式** | CSV 文件（timestamp, target_id, lat, lon） | Python list[dict]（实时内存对象） | 输入接口完全不同 |
| **数据预处理** | `prepare()` 函数：排序、差分、算速度/航向/岸边距离 | 这些数据已由 `radar_track_parser_v4.py` 实时提供 | 预处理逻辑重复 |

### 怎么改才能搬进来？

不需要搬整个文件。**把三段检测逻辑的核心判定条件，移植到当前 `anomaly_detector.py` 的对应方法中**：

| 搬什么 | 搬到哪 | 改动量 |
|--------|--------|--------|
| **搭靠**：相对速度约束 + 接近窗口距离下降量 | `_check_rendezvous()` 增加两个判定 | +30行 |
| **靠岸**：运动特征路径（接近速度+距离下降+到达后低速） | `_check_shore_docking()` 增加路径B | +40行 |
| **去重**：靠岸目标不再报快速通过 | `feed()` 末尾增加去重逻辑 | +15行 |
| **不搬**：pandas依赖、经纬度转换、`prepare()` 预处理 | — | — |

改动后 `anomaly_detector.py` 同时拥有"区域判断"和"运动特征判断"两条路径，区域未标定时也能靠运动特征检测。

## 七、配置说明

`live_config.json` 中需要根据实际水池调整的：

**区域定义**（`zones` 段）：
- `normal_channel`：正常航道的多边形顶点
- `rendezvous_zone`：搭靠检测区
- `fast_passage_zone`：快速通过检测区
- `unauthorized_docking_zone`：非正规靠岸区
- `authorized_docking_point`：正规停靠点（此处停靠不报警）

**检测阈值**（`anomaly_detection` 段）：
- 各项阈值见配置文件中的 `_comment` 注释

## 八、已知局限与改进计划

| 优先级 | 问题 | 计划 |
|--------|------|------|
| 🔴 P0 | 搭靠检测只看距离，没有判断"两船是否在靠近" | 加入相对速度和距离下降量判断 |
| 🔴 P0 | 靠岸检测完全依赖区域定义，区域未标定时无法工作 | 增加运动特征判断路径（只依赖岸边参考点） |
| 🔴 P0 | 同一目标可能被重复报两种异常 | 增加检测去重逻辑 |
| 🟡 P1 | 相机切换是被动的（目标超出范围才切） | 加入预测式切换（提前1秒判断） |
| 🟡 P1 | 没有跟踪质量评估 | 加入"当前相机能否跟得上"的判断 |
| 🟢 P2 | 没有俯仰(tilt)控制 | 后续迭代加入 |
| 🟢 P2 | 目标跟踪用最近邻，交叉时可能ID互换 | 后续考虑卡尔曼滤波 |

## 九、快速排查

| 现象 | 可能原因 | 检查方法 |
|------|----------|----------|
| 没有报警 | 区域坐标是否正确 | 对比水池实际布局和配置中的多边形顶点 |
| 误报太多 | 阈值偏小 | 调大 `distance_threshold_m`、`duration_threshold_s` 等 |
| 漏报 | 阈值偏大或冷却太长 | 调小阈值，缩短冷却时间 |
| 搭靠不报 | 两船不在搭靠区内 | 确认 `rendezvous_zone` 多边形覆盖了实验区域 |
| 靠岸不报 | 区域未标定 | 标定 `unauthorized_docking_zone` 或等待P0运动特征路径实现 |
| CSV为空 | `anomaly_detection.enabled` 是否为 `true` | 检查配置文件 |
