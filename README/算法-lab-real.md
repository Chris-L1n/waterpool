## 一、SC 船-船搭靠

### 实验室版（当前代码）

位置：`anomaly_detector.py` `_check_rendezvous()` 第 275-367 行

依赖：`radar_track_parser_v4.py` `SimpleTracker` 提供 `smooth_x_m/smooth_y_m`（EMA 平滑坐标，每4帧收敛）

```
每帧两两配对，双路径判断：

前置过滤（仅首次创建pair时检查一次）：
  双方最近50帧中速度>0.1的帧占比 ≥25% → 允许参与SC
  （排除墙壁/固定反射，通过后不再重复检查。pair的计时器跨帧保持不丢失）

路径1 · 距离阈值：
  EMA平滑距离 < 0.7m → 计时 → 满 5 秒 → 报警

路径2 · 距离稳定性：
  回溯 4 秒两船距离历史：
    标准差 < 0.18m → 距离波动极小（两船同动/绑定）
    平均距离 < 2.0m → 在同一集群内
  满足 → 有效阈值放宽至 max(0.7, 平均值×1.15)
  实际距离 < 放宽阈值 → 计时 → 满 5 秒 → 报警

触发类型：
  "distance"         — 路径1
  "distance_stable"  — 路径2

设计意图：
  路径1 覆盖两船贴到 0.7m 内的场景
  路径2 雷达报不准近距离时利用距离稳定性辅助判断
  （遥控船30cm+反射点偏差，紧靠时雷达报1~2m但波动极小）
```

| 参数 | 值 | 位置 |
|------|-----|------|
| `distance_threshold_m` | 0.7m | `live_config.json` |
| `duration_threshold_s` | 5.0s | `live_config.json` |
| EMA 平滑系数 | 0.25 | `SimpleTracker.__init__` |
| 距离稳定性窗口 | 4.0s | 代码内常量 |
| 稳定标准差上限 | 0.18m | 代码内常量 |
| 稳定平均距离上限 | 2.0m | 代码内常量 |

（速度差/接近窗口/距离下降量 全部已禁用，999）

----------------------------------------------------------------------

### 生产环境版

**全部恢复。**

| 参数 | 生产值 | 原因 |
|------|--------|------|
| `duration_threshold_s` | **15.0** | 实验方案文档要求 |
| `distance_threshold_m` | **2~5m** | 真实船舶尺寸大 |
| `relative_speed_threshold_m_s` | **1~3m/s** | 恢复启用。海面墙-船场景不存在，但浪面漂移物可以区分 |
| `approach_distance_threshold_m` | **10~50m** | 港口尺度 |
| `min_distance_drop_m` | **5~20m** | 港口尺度 |

**实现方式：** 只需把 JSON 中的四个 999 值改回真实值。代码无需修改。

## 三、KA 船靠岸送人上岸

### 实验室版（当前代码）

位置：`anomaly_detector.py` `_check_shore_docking()` 第 440-600 行

双路径，任一触发即报警：

**路径A（偏航+减速停留）：**

```
水池太小，不考虑授权点

① was_moving=True（必须曾经移动过 → 排除墙壁固定反射）
② X坐标 偏离航道中心线（normal_channel） >0.2m（偏航）
③ （速度检查已禁用，回到中心线即清零）//速度 <0.2m/s（减速停止） 
④ 持续 ≥ 3 秒 → 报警
```

**路径B（岸边停留，简化版）：**

```
  水池太小，不考虑授权点

  ① was_moving=True（曾移动过）
  ② 距 unauthorized_docking_zone 边界线 <0.5m → 进入 Y<1.0m 区域
  ③ 持续 ≥ 3 秒 → 报警
  （去掉了回溯/接近速度/距离下降等港区级别判断）
```

### 生产环境版

**可以去掉 `was_moving` 保护**（海面无固定反射）。路径A 逻辑不变，路径B 的坐标系改为 GPS。参数放大到港口尺度：

路径A（偏航+减速停留）
路径B（运动特征，不依赖区域多边形）


| 参数 | 生产值 |
|------|--------|
| `deviation_threshold_m` | **10~50m** |
| `speed_threshold_m_s` | **0.5~2m/s** |
| `duration_threshold_s` | **10~60s** |

## 四、滤波算法

### 实验室版（当前代码）

滤波分两层，串联执行：

#### 第一层：SimpleTracker（ID 跟踪 + EMA 平滑）— `radar_track_parser_v4.py`

```
输入：雷达每帧输出的原始目标
输出：带 track_id + smooth_x_m/smooth_y_m 的目标列表

① 最近邻匹配：
  当前帧目标在活跃轨迹中找距离最近的
  距离 < 0.4m → 继承旧 ID；找不到 → 分配新 ID

② 轨迹重连：
  目标消失 > max_missed(20帧=1.0s) → 移入"最近丢失列表"
  新目标出现 → 优先在丢失列表中匹配（距离放宽至0.8m）
  匹配成功 → 继承旧 ID（同一艘船短暂丢失后找回）

③ EMA 坐标平滑：
  每帧对活跃轨迹做指数移动平均（alpha=0.25，约4帧收敛）
  输出 smooth_x_m/smooth_y_m，SC算法使用
```

#### 第二层：BoatTargetSelector（船只过滤）— `boat_target_filter.py`

```
输入：SimpleTracker 输出的带 track_id 的目标列表
输出：过滤后的"船只目标"，交给异常检测器 + VOFA+

6 个过滤条件（从配置读取，null=不启用）：

① XY 位置范围
   x_min_cm=-120, x_max_cm=120  → 水池宽度1.2m + 雷达偏移余量
   y_min_cm=0, y_max_cm=400      → 水池长度 4m
   超出范围 → 丢弃（墙壁/天花板反射）

② PV 信号质量 ≥ 30
   弱信号 → 丢弃

③ 匹配距离 < 0.15m
   当前帧位置与上一帧匹配位置的偏差 >0.15m → 丢弃（抖动噪声）

④ 最大速度 < 200cm/s
   雷达偶尔跳到 11m/s，200cm/s 排除极端毛刺

⑤ 最小总位移 ≥ 0.3m（最强过滤）
   目标从首次出现到现在，总位移 <0.3m → 丢弃
   等待 15 帧(1.5s)后才判断，避免误杀慢启动的船
   墙壁反射位移 ~0.03m vs 船 2m+

⑥ 最小存续帧数 ≥ 10 帧
   短命目标 → 丢弃（排除偶然噪声）
```

#### 锁轨机制

```
lock_track=true：锁定后不轻易切换目标
  - 从候选船中按评分函数选最佳目标（hits×100 + PV×0.2 - speed×0.01 - distance×0.1）
  - 锁定目标持续存在 → 始终选它
  - 锁定目标消失 > lock_stale_packets(15帧=0.75s) → 释放锁，重新选择
```

#### 实时运行流程 — `radar_live.py`

```
CAN帧 → 雷达组包 → parse_target_packet() → 原始目标列表
  ↓
SimpleTracker.update() → 分配 track_id
  ↓
BoatTargetSelector.select() → 过滤后选一条船作为 nearest
  ↓
boat_targets = 所有通过过滤器的目标（可能多条船）
  ↓
AnomalyDetector.feed(boat_targets) → SC/KS/KA 检测
  ↓
VOFA+ UDP 转发 boat_targets → 实时点云可视化
```
