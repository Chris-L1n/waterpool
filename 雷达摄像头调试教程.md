# 水池雷达与三摄像头实时联动系统

该程序用于水池实验：CAN 雷达检测真实船只的位置，程序选择合适的乐橙摄像头，并通过开放平台 HTTP 接口控制云台水平转动，同时保存雷达目标和轨迹数据。

## 工作流程

```text
真实船只移动
    ↓
雷达通过 CAN 输出目标
    ↓
解析 X、Y、速度和 PV
    ↓
分配稳定 track_id，并选择最近目标
    ↓
根据位置和覆盖范围选择摄像头 A、B 或 C
    ↓
计算目标相对摄像头的水平角度
    ↓
调用乐橙 controlPTZ 接口
    ↓
摄像头水平转动并记录雷达 CSV
```

## 文件说明

| 文件 | 作用 |
| --- | --- |
| `main.py` | 唯一需要运行的程序入口 |
| `live_config.json` | 雷达、摄像头位置、云台控制和开放平台配置 |
| `radar_track_parser_v4.py` | CAN 雷达分包重组、目标解析、Track ID 和 CSV 输出 |
| `radar_live.py` | 将雷达解析器封装成实时目标数据源 |
| `live_tracking.py` | 目标选择、三摄像头接力和云台指令调度 |
| `imou_ptz.py` | accessToken、签名和 controlPTZ 请求 |

除了 `ControlCAN.dll`，实时模式只使用 Python 标准库，不需要安装 NumPy、Matplotlib 或 OpenCV。

## 实验前准备

1. 三台摄像头完成配网，在乐橙 App 和开放平台中保持在线。
2. 电脑可以访问互联网，以调用乐橙开放平台。
3. 雷达连接到 CANalyst-II，CANalyst-II 连接电脑。
4. 关闭 USB-CAN Tool，避免它占用 CAN 设备。
5. 准备与 Python 位数匹配的 `ControlCAN.dll`。

## 第一步：配置三台摄像头

打开 `live_config.json`。

每台摄像头必须配置：

```json
{
  "key": "A",
  "device_id": "D35C8CCPCGA7C1E",
  "position_m": [0.0, 0.0],
  "heading_deg": 0.0,
  "initial_pan_deg": 0.0,
  "max_range_m": 150.0
}
```

参数含义：

- `position_m`：摄像头在雷达坐标系中的 `[X, Y]` 位置，单位米。
- `heading_deg`：摄像头处于预设位置时，正前方相对雷达 `+Y` 方向的夹角。
- `initial_pan_deg`：程序启动时摄像头相对预设正前方的水平角度。
- `max_range_m`：摄像头能够用于跟踪的最大距离。

当前配置中的 `[0,0]、[150,0]、[300,0]` 来自旧仿真，不能直接用于真实水池。必须测量三台摄像头相对雷达的实际位置后修改。

坐标约定：

```text
雷达正前方：+Y
雷达右侧：+X
雷达左侧：-X
```

## 输出数据

程序会在项目目录生成：

| 文件 | 内容 |
| --- | --- |
| `radar_targets.csv` | 每个雷达包中的全部目标 |

| `radar_tracks.csv` | 带稳定 `track_id` 的全部轨迹点 |

## 联动控制参数

`live_config.json` 中：

- `deadband_deg`：水平误差小于该角度时不转动，避免抖动。
- `max_step_deg`：单次接口最多调整的角度。
- `command_interval_s`：两次云台请求之间的最小时间。
- `handoff_cooldown_s`：切换摄像头后的冷却时间。
- `target_timeout_s`：超过该时间没有新目标时停止产生新指令。
- `ms_per_degree`：角度到 duration 的换算，目前为实验值 `26.4ms/度`。

## 当前范围和限制

1. 摄像头云台目前只根据雷达控制水平方向。
2. 垂直方向应结合视频目标框的位置控制，雷达平面 X/Y 无法直接给出可靠的俯仰误差。
3. 程序当前控制通过过滤条件的最近目标，并把所有目标写入 CSV。
4. 云台接口没有返回真实角度，程序依靠指令累计估计当前水平角度，长时间运行可能产生误差。
5. 正式实验前应让三台摄像头回到统一预设位置，再启动程序。

## 常见问题

### 找不到 ControlCAN.dll

确认 DLL 已放入项目目录，并确认 Python 与 DLL 位数一致。

### 打不开 CAN 设备

关闭 USB-CAN Tool 和其他雷达程序，拔插 USB-CAN 后重试。

### 雷达有目标但没有云台命令

检查目标是否通过 `filters`，并确认目标位于至少一台摄像头的 `max_range_m` 范围内。

### 只打印 DRY RUN

这是安全模式。完成标定后将 `dry_run` 改成 `false`。

### 接口返回成功但摄像头不转

检查摄像头是否在线、是否到达机械边界、设备序列号是否正确，以及开放平台账号是否具有该设备的控制权限。

# 乐橙摄像头 PTZ 水平与垂直控制说明

## 1. 文件用途

当前正式联动项目使用下面这个脚本测试和调用云台：

```text
D:\27610\Documents\目标识别\test_3_integrated\imou_ptz.py
```

该脚本通过乐橙开放平台 `controlPTZ` 接口控制三台摄像头，支持以下动作：

| action | 作用 |
| --- | --- |
| `move_left` | 向左转动 |
| `move_right` | 向右转动 |
| `move_up` | 向上转动 |
| `move_down` | 向下转动 |
| `left_limit` | 持续向左移动到参考边界 |
| `right_limit` | 持续向右移动到参考边界 |
| `home` | 先向左到参考边界，再向右回到预设位置 |
| `stop` | 立即停止云台 |

## 2. 运行位置

先进入项目根目录：

```powershell
cd D:\27610\Documents\目标识别
```

然后使用下面这种格式运行：

```powershell

```

如需查看完整请求体和返回结果，在命令末尾加 `--verbose`：

```powershell
python .\test_3_integrated\imou_ptz.py --camera A --action move_right --degrees 10 --verbose
```

## 3. 摄像头编号

| 摄像头 | camera 参数 | 设备序列号 |
| --- | --- | --- |
| 摄像头 A | `A` | `D35C8CCPCGA7C1E` |
| 摄像头 B | `B` | `D35C8CCPCGC5BC1` |
| 摄像头 C | `C` | `D35C8CCPCG2DF8B` |

## 4. 参数说明

| 参数 | 说明 | 可用值/默认值 |
| --- | --- | --- |
| `--camera` | 选择摄像头 | `A`、`B`、`C`，默认 `A` |
| `--action` | 选择云台动作 | 见动作表，默认 `move_right` |
| `--speed` | 云台移动速度 | `1` 至 `8`，默认 `8` |
| `--degrees` | 普通移动角度 | 默认 `30` |
| `--limit-degrees` | 到达左右参考边界时的等待量 | 默认 `360` |
| `--home-degrees` | 从左参考边界向右返回的角度 | 默认 `180` |
| `--verbose` | 打印 OpenAPI 请求体和返回结果 | 可选 |

脚本使用以下换算关系：

```text
1 度 = 26.4 毫秒
```

例如：

| 角度 | duration |
| --- | --- |
| 10 度 | 264 ms |
| 30 度 | 792 ms |
| 45 度 | 1188 ms |
| 90 度 | 2376 ms |
| 180 度 | 4752 ms |
| 360 度 | 9504 ms |

## 8. 垂直控制注意事项

当前正式脚本的垂直控制请求体中 `h=0`，不会主动加入水平速度。

但是需要注意：

1. 垂直动作仍然使用 `duration="last"`，随后固定等待约 2 秒再停止。
2. `--degrees` 在垂直控制中主要用于保持命令格式一致，不一定等价于真实垂直角度。
3. `1度 = 26.4ms` 是水平运动测量结果，不一定适用于垂直方向。
4. 垂直方向如果要做精确实验，需要单独标定。

## 9. 常见问题

### 返回重复 nonce

脚本每次请求都会使用 UUID 生成新的 `nonce`。如果仍返回重复 nonce，重新运行脚本，并确认没有复制固定请求体反复发送。

### 返回权限不足

确认设备已经绑定到对应开放平台账号，并确认应用具有云台控制权限。

### 摄像头没有移动

依次检查：

1. 摄像头是否在线；
2. 设备序列号是否正确；
3. `channel_id` 是否为 `0`；
4. 云台是否已经到达机械边界；
5. 上一个 `duration="last"` 动作是否已经停止；
6. 终端返回的 `result.code` 是否为 `0`。

### 如何强制停止

```powershell
cd D:\27610\Documents\目标识别
python .\test_3_integrated\imou_ptz.py --camera A --action stop
```

将 `A` 替换为 `B` 或 `C`，即可停止对应摄像头。
